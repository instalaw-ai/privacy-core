# Security Policy

## Reporting a vulnerability

If you discover a security vulnerability in `@instalaw/privacy-core`, please report it through our coordinated disclosure form:

**https://alpha.security.instalaw.io**

Do **not** open a public GitHub issue for security vulnerabilities — doing so may expose users before a fix is available.

### What to include

- A description of the vulnerability and its potential impact.
- Steps to reproduce, ideally with a minimal proof-of-concept.
- Affected version(s) and, if known, the affected module (`crypto`, `auth`, `redaction`, `byom`, or `utils`).
- Your preferred contact method for follow-up.

### What to expect

- We acknowledge reports within three business days.
- We keep you informed as we investigate and prepare a fix.
- Upon release, we credit you in the release notes unless you request otherwise.

## Supported versions

Only the latest minor release on npm receives security patches. Pin to a recent version to stay covered.
